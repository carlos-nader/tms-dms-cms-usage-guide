SECTION 4.2.2 STRUCTURE (FINAL) text 4.2.2 DMS Down: Toggle SOI Between Displays [Intro — unnumbered, no \subsubsection] 4.2.2.1 DMS Down Effectiveness in All Master Modes 4.2.2.2 DMS Down Usage Table (hotastable) 4.2.2.3 DMS Down Exception States INTRO CONTENT (Unnumbered, ~2–3 Paragraphs) Opening concept + Toggle vs Cycle clarification:

DMS Down toggles the Sensor of Interest (SOI) between the HUD and the two Multifunction Displays (MFDs), allowing the pilot to move SOI responsibility among these displays hands-on. The exact behavior depends on which master mode is active and whether the HUD is a valid SOI candidate in that mode (see Section 4.1.1).

In master modes where the HUD is a valid SOI (NAV, A-G PRE, A-G VIS), repeated presses of DMS Down cycle through a repeating sequence: HUD → right MFD → left MFD → HUD → ... This allows the pilot to move SOI among all three displays.

In master modes where the HUD is not a valid SOI candidate (A-A, DGFT, MSL OVRD), DMS Down toggles SOI only between the two MFDs (right MFD ↔ left MFD), and the HUD remains a passive display regardless of SOI status. This allows the pilot to choose which MFD sensor receives hands-on HOTAS commands.

Clarification note:

The distinction between "toggle" and "cycle" is a consequence of the avionics architecture's constraints on which displays can be SOI in each mode (detailed in Section 4.1.1). The DMS Down mechanism itself is uniform across all modes: it simply advances SOI to the next valid candidate in the system's SOI set for that master mode. [Note: This reference to Section 4.1.1 is a forward/backward cross-ref; it clarifies the toggle concept without duplicating future section 4.2.3 details about DMS Left/Right.]

4.2.2.1 DMS DOWN EFFECTIVENESS IN ALL MASTER MODES Structure: Follow the same pattern as 4.2.1.1 (DMS Up):

Master Modes Where HUD Is a Valid SOI Candidate (NAV, A-G PRE, A-G VIS) NAV (Navigation) Master Mode DMS Down cycles through all valid SOI candidates in NAV: HUD, FCR, TGP, HSD, WPN, HAD. In practice, the physical layout (HUD + left/right MFDs) creates a repeating 3-step cycle:

HUD SOI → press DMS Down → right MFD SOI (FCR or TGP, depending on format configuration) → press DMS Down → left MFD SOI (HSD or WPN, depending on format configuration) → press DMS Down → HUD SOI → ...

This allows the pilot to move hands-on command focus among the HUD (for heading/altitude/situational awareness) and the two MFD sensor pages (for detailed navigation sensor management). DMS Down is a convenience tool in NAV, enabling quick SOI transitions without OSB manipulation.

A-G PRE (Preplanned Air-to-Ground) Master Mode In A-G PRE, valid SOI candidates are HUD, FCR, TGP, WPN, HAD, HSD. DMS Down cycles through the same pattern as NAV:

HUD SOI → right MFD SOI → left MFD SOI → HUD SOI → ...

However, the specific sensors on each MFD depend on how the pilot has configured the displays (e.g., right MFD might show TGP, left MFD might show HSD or WPN). DMS Down moves SOI among these configurations without changing which format is displayed. Like NAV, DMS Down in A-G PRE is a convenience tool, allowing the pilot to shift hands-on control between HUD and MFD sensor pages during mission planning or setup phases.

A-G VIS (Visual Air-to-Ground: CCIP, DTOS, AGM-65 VIS, IAM-VIS, HUD/HMCS MARK) In A-G VIS modes, valid SOI candidates are HUD, FCR, TGP, WPN. DMS Down again cycles through the 3-step pattern:

HUD SOI → right MFD SOI (typically TGP or WPN) → left MFD SOI (typically FCR or HSD) → HUD SOI → ...

However, in A-G VIS, DMS Down is operationally critical, not merely convenient. This is because A-G VIS designations are HUD/HMCS–centric: the pilot aims using the HUD pipper (CCIP), target designator (AGM-65 VIS, IAM-VIS), or HUD/HMCS MARK symbology. These visual cues are controlled by CURSOR/ENABLE and TMS inputs routed through HUD SOI. If SOI migrates to an MFD (TGP, WPN, FCR, HSD), TMS and CURSOR commands are routed to that MFD display instead, and the HUD visual cueing no longer responds as expected.

Therefore, DMS Down is the critical complement to DMS Up in A-G VIS flows:

DMS Up re-establishes HUD SOI when SOI has "drifted" to an MFD during sensor management or configuration.

DMS Down allows the pilot to temporarily move SOI off the HUD and onto an MFD (TGP for refining sensor search, WPN for weapon status, FCR/HSD for situational awareness outside the HUD field of view), then alternate back to HUD to resume visual designation.

In A-G VIS, this Up/Down alternation is a fundamental part of the delivery flow and cannot be omitted without losing command flow efficiency or situational awareness.

Master Modes Where HUD Is NOT a Valid SOI Candidate (A-A, DGFT, MSL OVRD) In air-to-air employment and dogfight modes, the avionics architecture restricts SOI to sensor formats on the MFDs: FCR, HSD, and TGP. The HUD cannot be SOI in these modes (see Section 4.1.3 for the architectural rationale).

Consequently, DMS Down is restricted to toggling SOI between the two MFDs only:

right MFD SOI ↔ left MFD SOI.

This 2-way toggle allows the pilot to choose which MFD sensor page receives hands-on command priority. In A-A contexts, this is operationally primary: the pilot uses DMS Down to alternate SOI focus between, for example, FCR (for track/lock/missile control) and HSD (for picture/sorting/threat assessment), or between FCR and TGP (when using the targeting pod for situational awareness or supplemental tracking).

DMS Down is the standard HOTAS mechanism for managing SOI in A-A, DGFT, and MSL OVRD modes, and its use is essential to efficient radar employment and tactical decision-making.

Snowplow (SP) PRE State (Unstabilised) When the pilot enters Snowplow mode (a specialised A-G ground-stabilisation mode for slewing to arbitrary ground positions) and the SP position has not yet been stabilised with TMS Up, the SOI is effectively "nowhere". Both the A-G radar and TGP MFD displays show NOT SOI, and neither display is designated as SOI. As a result, DMS Down has no effect in this state: the toggle mechanism has nowhere to advance SOI to.

Once the SP position is stabilised with TMS Up (pressing TMS Up on the HUD), SOI returns to its previous designated display, and DMS Down resumes normal toggling behaviour.

MARK/OFLY Submode In the MARK/OFLY submode (a specialised target-acquisition context documented in Dash-34 §2.1.1.2.3), the SOI cannot be designated or changed at all. Consequently, DMS Down has no effect in MARK/OFLY: you cannot toggle SOI when SOI designation itself is locked. This submode is rare in normal operations but is important to recognise if you encounter it during unusual procedures or system states.
